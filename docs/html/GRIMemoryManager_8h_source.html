00033 // A memory management class
00034 //
00035 // The GRIMemoryManager class is meant to be used as a library object. That is to
00036 // say that all other thread objects will be using the memoryManager class's
00037 // functions in order to access memory buffers. At the start of the data
00038 // aquisition process, exactly one memoryManager class should be created and
00039 // accessed by all other threads.
00040 //
00041 // the purpose of the GRIMemoryManager are as follows:
00042 // 1. to create and manage a memory space for reading and writing buffers.
00043 // 2. to synchronize read and writes between different threads to buffers
00044 // 3. to prevent overflow of buffers and the virtual memory address space
00045 // 4. to act as an abstraction layer to the software's memory
00046 //
00047 // Elements in the GRIMemoryManager should in most cases not be directly accessed.
00048 // All internal variables will be locked as private and can only be accessed 
00049 // through memory and thread safe functions. In cases where data needs to be
00050 // returned, only a copy of the data will be handed off rather than the pointer
00051 // or original data for the sake of memory security.
00052
